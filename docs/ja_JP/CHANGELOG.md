## [2.3.1] - 2026-07-07

### Changed

* アプリの設定の保存・検証の仕組みを刷新し、設定をより分かりやすく確実に (#260)
  * 設定値が有効な範囲で確実に検証されるようになり、範囲外・不正な値は適用されずに自動修正または拒否
  * エラーで「何がダメか（範囲外／整数が必要）」と有効な範囲を入力欄のすぐ下に表示する
* 設定の保存・読み込みの内部処理を改善し、設定へのアクセスをわずかに高速・効率化 (#261)
* 内部コードを整理し、タグ・シリーズの更新処理を効率化。あわせて更新進捗ダイアログの軽微な余白の不具合を修正 (#279)
* 読書中のパフォーマンスを改善し、ページ送りでライブラリ全体を再読み込みしないように (#283)
* フロントエンドのエラーコードを Rust バックエンドから自動生成し、常に同期が取れるように改善 (#287)

### Fixed

* フォルダーの更新監視が有効なときに、フォルダーをすばやく切り替えると古い監視がバックグラウンドに残り、リソースを浪費したり、意図しない一覧の更新が発生する場合がある不具合を修正 (#262)
* ライブラリと設定に関するデータ整合性の不具合を修正 (#264)
  * ディスク上でファイルが変わった本（例: フォルダーへのページ追加）でページ数やタイトルが更新されるようになり、追加したページを開けるように修正
  * 設定ファイルの値が1つでも不正だと全設定が初期値に戻る問題を修正し、該当する設定だけを初期値に戻して他は保持するように修正
  * 同じタグを複数回選んだ状態でも本へのタグ設定が失敗しないように修正
  * 異なるフォルダーに同名の画像を含む一部の固定レイアウトEPUBで、ページが誤った順序で表示されないように修正
* 高さの大きい透過画像（透過PNGなど）をリサイズすると透過部分が背景色で塗りつぶされる不具合を修正し、透過を保持するように修正 (#265)
* リーダーと本棚の細かな不具合を修正 (#266)
  * 前回読んだあとにページが削除されていると、本が空白ページで開いてしまう場合がある不具合を修正し、利用可能な最後のページで開くように修正
  * 特定のタイミングでアップデート用ダイアログを閉じると、アプリを再起動するまで以降のアップデート確認がすべて行えなくなる、まれな不具合を修正
  * 本棚を読み込み中に開くと、読書中の本までスクロールされない場合がある不具合を修正し、一覧の読み込み完了後に読書中の本へスクロールするように修正
* 本棚を日付順にすると単体の本が登録順で並んでいた不具合を修正し、本とシリーズが実際の日付で混在して並ぶように修正 (#267)
* 本棚のグリッドの左側の余白部分でスクロールできない不具合を修正し、全体でスクロールできるように修正 (#268)
* リーダー・本棚・ファイル処理の複数の不具合を修正 (#269)
  * 読み込みに失敗した本を開いたあとも、前に開いていた本のページが表示され続ける不具合を修正
  * 読み取れない項目（壊れたショートカットなど）を含むフォルダーが空に見える不具合を修正し、残りの項目は引き続き一覧表示するように修正
  * 同名の画像を2つ含むコミックアーカイブ（ZIP）で、1ページが二重表示され別のページが表示されない不具合を修正
  * サイズ情報が不正なコミックアーカイブ（ZIP）でページを開くとアプリがクラッシュする場合がある不具合を修正し、安全に処理するように修正
  * 大きな本を開く際に、画像読み込みなど他の操作が一時的にブロックされる不具合を修正
  * SVGファイルが読み込み可能なページとして一覧表示されるものの常に表示に失敗する不具合を修正し、ページとして表示しないように変更
  * 選択したタグの一つがその間に削除されていても、本へのタグ設定が失敗しないように修正
  * 開いているフォルダー内の細工されたファイル名で、フォルダー外のファイルを読み取れるパストラバーサルの問題を修正
* リーダーのページ移動に関する不具合を修正 (#270)
  * 見開き表示で、レイアウトが変わった直後に前送りすると移動量を誤り、ページを飛ばす・重複する場合がある不具合を修正し、前送りが現在のページのレイアウトに従うように修正
  * 見開き表示で、単ページと見開きが混在するなかで前に戻ると見開きの途中に着地する場合がある不具合を修正し、正しいページへ戻るように修正
  * 見開き表示で、見開きの2ページ目の読み込みに失敗するとビューアが空白または前のページのままになる場合がある不具合を修正し、表示できるページにフォールバックするように修正
  * ファイルナビゲーターでファイルをダブルクリックしても何も起きない不具合を修正し、ファイルを開くように修正
  * 表示中の本を選び直すと、保留中の「最初／最後のページで開く」操作が後続の無関係な本に持ち越される場合がある不具合を修正
* 本棚の複数の不具合を修正 (#271)
  * 本棚で、検索により一覧が短くなった直後に Enter または Space を押すとグリッドがクラッシュする場合がある不具合を修正
  * 本棚を開く際、グリッド幅が計測される前に読書中の本への自動スクロールが誤った行に着地する場合がある不具合を修正し、幅が確定してからスクロールするように修正
  * シリーズ内の本を並び替えても本棚（シリーズの表紙スタックやシリーズ内の並び）に即座に反映されない不具合を修正し、すぐに反映されるように修正
  * 最初のページだけ読んだ本の進捗が実際の読書進捗ではなく0%と表示される不具合を修正し、進捗バーに正しく反映するように修正
* 設定とレイアウトの複数の不具合を修正 (#272)
  * PDFの描画解像度の入力欄を空にすると、必ず検証に失敗する不正な値が保存される不具合を修正し、現在の値を維持するように修正
  * 保存されたサイドペインのタブが存在しなくなった際、最初のタブへのリセットで設定が繰り返し保存され続ける不具合を修正
  * ページを変えた直後に「読書履歴を記録」をオフにしても、その最後のページが記録される場合がある不具合を修正し、遅延書き込みの直前に設定を再確認するように修正
  * ペインのサイズを変更した直後に画面を切り替えたり閉じたりすると、最後のサイズが破棄される不具合を修正し、最新のサイズを保存するように修正
* ファイルブラウザーを日付で並べ替えると、月初表記ではない地域の日付形式で並び順が誤る場合がある不具合を修正し、システムのロケールに関わらず正しく並ぶように修正 (#273)
* 別ウィンドウで設定を変更した場合（または保存時に拒否された場合）、開いている設定ウィンドウの一部のトグルやドロップダウンが古い値を表示し続ける不具合を修正し、常に保存済みの設定を反映するように修正 (#274)
* 一部のEPUBで、読み取れないチャプターが1つあると本の残りのページが読み順から外れる場合がある不具合を修正し、残りのページが正しい順序を保つように修正 (#275)
* 本は正常に開けているのに、バックグラウンドのページ先読みに失敗すると本を開く操作自体が失敗扱いになる場合がある不具合を修正し、先読みをbest-effortとして本を開く処理を失敗させないように修正 (#276)
* ファイルナビゲーターで、異なる2つの行を素早くシングルクリックすると2番目の行のダブルクリック（ファイルを開く・フォルダーに入る）と誤認される不具合を修正し、各行が選択されるだけになるように修正 (#277)
* 設定の保存に失敗した際に、誤って「入力内容を確認してください」と表示される不具合を修正し、「設定の保存に失敗しました」と明確に表示するように修正。また入力検証の失敗時は汎用エラーではなく適切な案内を表示するように修正 (#278)
* 本を別のシリーズへ移動すると移動先のシリーズ内で並び順が乱れる場合がある不具合を修正し、シリーズ間で移動した本が末尾に正しく配置されるように修正 (#280)
* コミック・PDF・EPUB の処理に関する複数の不具合を修正 (#281)
  * 一部の固定レイアウトEPUBでページが読み順どおりに表示されない不具合を修正し、本のspine（読み順）に従って表示するように修正
  * 変わった名前（非Unicode）のファイルを含むフォルダーを開くと全体が失敗する場合がある不具合を修正し、読み取れないファイルはスキップして残りのページを開くように修正
  * 一部のRARアーカイブで1ページが二重表示され別のページが表示されない不具合を修正し、重複エントリを正しく処理するように修正
  * PDFのサムネイルが過大・不鮮明になる場合がある不具合を修正し、他の形式と同じサムネイルサイズに揃えるように修正
  * 不正な「zip爆弾」アーカイブでメモリを枯渇させ得るコミックアーカイブ（ZIP）の処理を強化
  * 低速なストレージ上で大きな本を開く際に、他の操作が一時的に固まる不具合を修正
  * 本をすばやく切り替えると前の本のページが一瞬表示される場合がある不具合を修正し、そうした古いページ要求を拒否するように修正
* 設定ファイルが一時的にロックされている・読み取れない場合（アンチウイルスやバックアップツールなど）に設定が初期値へリセットされる不具合を修正し、現在の値で動作しファイルをそのまま保持するように修正 (#282)
* 本をすばやく切り替えると直前の本の最終ページ位置が失われる場合がある不具合を修正し、切り替え前に直前の本の位置を保存するように修正 (#283)
* リーダーの複数の不具合を修正 (#284)
  * 最初の本の読み込み中に2冊目を開くと誤った本が表示され、その本に読書進捗が保存される場合がある不具合を修正し、新しく開いた本が常に優先されるように修正
  * 見開き表示で、ページのレイアウトが判明する前に前送りするとページが恒久的に飛ばされる場合がある不具合を修正し、その場合は1ページだけ進むように修正
  * 別の本を開いたあとに同じファイルを再度ドラッグしても何も起きない不具合を修正し、ファイルを再度開くように修正
  * 長時間の読書で訪れた全ページがメモリに保持され続けないように改善し、リーダーのメモリ使用量を削減
* 本棚の複数の不具合を修正 (#285)
  * 本棚をすばやく切り替えると一瞬前の本棚の内容に戻る場合がある不具合を修正し、最新の選択が常に優先されるように修正
  * タグ設定・シリーズ設定・本棚追加のダイアログが無言で失敗する場合がある不具合を修正し、エラーを表示して部分的な変更を反映し、リトライのために開いたままにするように変更
  * ある本のタグダイアログを開いた直後に別の本で開くと誤った本のタグが表示・保存される場合がある不具合を修正し、常に現在の本を反映するように修正
  * プレーンクリックで本が開けない場合がまれにある不具合を修正し、確実に開くように修正
* EPUBアーカイブとレンダリングのスレッドプールのエラーが、専用のメッセージではなく汎用のメッセージで表示される不具合を修正 (#286)

## [2.3.0] - 2026-06-14

### Added

* 本の最後/最初のページを超えてページをめくると、次/前の本を自動で開く機能を追加（リーダー設定で「オフ / 確認してから開く / 自動で開く」を選択可能） (#254)

### Changed

* バックグラウンドでのページ先読み処理を最適化 (#247)
* 内部の状態管理を改善し、安定性と保守性を向上 (#248)
* 画像・コンテナ処理パイプラインの統合によるメモリ使用率を最適化、およびコミック（ZIP/RAR）のサムネイル生成速度を向上 (#249)
* 画像読み込みの高速化とメモリ使用量を削減、および本棚画面の描画パフォーマンスを向上 (#250)
* 読書進捗の保存処理における内部構造を改善し、アプリ全体の保守性を向上 (#251)
* 内部コードの型定義およびファイル構造をクリーンアップし、安定性とメンテナンス性を向上 (#252)
* 内部のエラー通知を改善し、リーダーの軽微なメモリリークを修正して、安定性と保守性を向上 (#255)

### Fixed

* シリーズ内の本のタイトルが短い場合に「シリーズの並び変え」ダイアログの横幅が不自然に狭くなる不具合を修正 (#245)
* 新しいシリーズやタグを作成した際に、「シリーズを設定」および「タグを設定」ダイアログが意図せず閉じてしまう問題を修正 (#246)

## [2.2.1] - 2026-06-01

### Changed

* 保守性と型安全性の向上のため、システム全体をDDD（ドメイン駆動設計）ベースのアーキテクチャとZodスキーマへ刷新 (#235)
* 読書進捗や本棚の変更が、開いているすべてのウィンドウへ即座に反映されるように改善 (#236)
* EPUBファイルをよりスムーズかつ安全に開けるよう、読み込み処理を最適化 (#237)

### Fixed

* アプリの設定（起動時のアップデート確認、画像のリサイズ方式、PDFの描画品質）が正しく保存・適用されない不具合を修正 (#234)
* 画面下部のスライダー（ページ進捗バー）を操作した後に、キーボードでのページめくりが1ページずつになってしまう不具合を修正 (#238)
* 本棚画面で検索ワードの入力、ウィンドウのリサイズ、または本棚情報が更新された際に、スクロール位置が不自然にリセット（ジャンプ）されてしまう不具合を修正 (#239)

## [2.2.0] - 2026-05-19

### Added

* 読書時のパフォーマンスを最適化するために画像キャッシュとページの先読み設定を追加 (#216)
* フォントをより素早く選択できるようにフォント設定に検索・フィルタリング機能を追加 (#219)
* シリーズ内の本をドラッグ＆ドロップで自由に並べ替えられる機能を追加 (#226, #229)
* 本棚で現在読んでいる本の強調表示と、その本への自動スクロール機能を追加 (#231)

### Changed

* 既にサムネイル画像が存在する場合は、コンテナ（ZIP/PDF等）の解析や画像抽出処理をスキップするように最適化し、処理速度を向上 (#214)
* 設定画面のレイアウトを整理・統合し、目的の設定を見つけやすいタブ構成に改善 (#217)
* サイドバーの表示状態（表示・非表示）および選択中のタブをアプリ設定として保存し、次回起動時に復元するように改善 (#223)
* シリーズ表示に対応した検索ロジックの改善と、シリーズ詳細への移動時に検索を自動リセットするように変更 (#227)
* 本の起動と画像処理の最適化により、読書開始までの待ち時間を短縮し操作性を向上 (#228)

### Fixed

* 本棚の仮想スクロールが全件描画される不具合を修正 (#215)
* 起動時の画面設定が正しく反映されない問題を修正 (#220)
* アプリのテーマ設定（ライト/ダーク）より、OSの設定が優先されてしまう不具合を修正 (#221)
* サイドパネルを非表示にした状態でアプリを終了すると、次回起動時にパネルの幅がリセットされる問題を修正 (#224)

## [2.1.0] - 2026-05-07

### Added

* コミックリーダーに、ショートカットキーやマウスボタンで表示を切り替えられるカスタマイズ可能なルーペ（拡大鏡）機能を追加 (#194)
* 本棚のグリッドに複数選択機能（Ctrl/Cmd+クリック、Shift+クリック）を追加し、複数の本に対して一括操作（タグ設定、コレクションへの追加、本棚からの削除）機能を追加 (#195)
* 本棚のタイトルの自動スクロールを切り替える設定を追加（オフにすることで動作を軽量化する） (#196)
* 本棚画面にシリーズのグループ化とドリルダウン（階層移動）機能を追加 (#200)

### Changed

* ファイル読み込みの高速化と、よりスムーズなページめくりを実現 (#193)
* CPUのSIMD命令とマルチスレッドを活用することで、画像のリサイズとサムネイル生成の速度を大幅に向上 (#199)
* UIの応答性を向上させるため、バックグラウンドでの画像読み込みスレッドの優先度を下げる (#204)
* コミックリーダーにおいて、ページの読み込みに時間がかかる場合にローディングを表示し、処理中であることが視覚的に分かりやすくなるよう改善 (#206)

### Fixed

* 画像リサイズ処理のパフォーマンスを向上させ、透過画像の縁に黒い枠が出る問題を修正 (#201)
* 読み込み効率の問題を回避するために PDF コンテナにおける画像の読み込みをシングルスレッドに制限 (#203)

## [2.0.3] - 2024-04-24

### Changed

* 本棚表示のレンダリングパフォーマンスを最適化 (#178)
* Reduxのセレクタ（`useAppSelector`）を `createSelector` を用いてメモ化し、不要なコンポーネントの再レンダリングを防止して描画パフォーマンスを向上 (#186)
* 画面左側のペイン（サイドバー）のサイズを保存・復元するロジックを、共通のカスタムフック `usePaneSizes` として切り出し (#189)
* 依存関係を更新 (#190)

### Fixed

* ImageLoader のスレッドプールサイズの計算を修正 (#177)
* 本棚画面で縦スクロールバーが表示・非表示される際に、内部のレイアウトが横にずれる問題を修正 (#179)
* AutoScrollTypography でスクロール開始までの遅延時間が反映されず、すぐにスクロールが始まってしまう問題を修正 (#180)
* 本棚画面において、最後のページまで読んでも読書進捗バーが100%にならない問題を修正 (#182)
* 本棚画面を開いた直後やビューを切り替えた際に、本のグリッドが一時的に不自然な幅で表示されてしまう（チラつく）問題を修正 (#183)
* 起動時の初期ビューの設定で、誤ったアイコンが表示されていた問題を修正（Language アイコンから RocketLaunch アイコンへ変更）(#184)
